DukuDukuChat
Social / Communication
One Pi Network app for your feed, stories, chats, channels & short videos.
DukuDukuChat brings the best parts of Facebook, WeChat, TikTok, Telegram and Instagram into a single app built on the Pi Network. Share posts and photos to a public or friends-only feed, post 24-hour stories, scroll short-form Discover videos, message friends and groups (with optional passcode-locked secret chats), broadcast to channels, and send Pi tips directly on posts or in a dedicated Pi Wallet. The centre navigation button is fully owner-customisable — set it to open the composer, start a video, launch a story, scan a QR code to connect, or open Pi payments. Sign in is Pi Network only, and every login is verified server-side against Pi's own API.

4 Mainnet Listing Requirements — updated status

Compared to the scorecard in DukuDukuChat_PiListing_Requirements.html, this pass closed two of the three open gaps.

  • ✓
    Pi-only authentication — done. DukuDukuChat_PiReady.html offers Pi Network sign-in only.
  • ✓
    Report / block moderation — done. Reporting posts, videos, and chats, plus blocking users (with automatic feed filtering), is fully wired front-to-back.
  • ✓
    Privacy policy — written. Still needs to be hosted at a public URL you control (a static file host, or served by the same backend).
  • •
    KYC, deployment, Portal registration, domain verification — unchanged from before: these are account-level actions only you can complete.
The moderation and Pi-only-login code has been syntax-checked and integration-tested against a faithful mock of the real API contract (see server/README.md for why this sandbox can't run a real npm install) — but it has not been tested against the real Pi Network, real deployed infrastructure, or real user reports. Treat your first deployment as a real smoke test, the same caveat as the rest of the backend.

5 Your remaining steps, in order

  1. Deploy server/ (Render/Railway/your own host — see server/README.md) and run npm install for real, treating first boot as a smoke test.
  2. Host DukuDukuChat_PrivacyPolicy.html at a public URL and update the support email inside it.
  3. Point DukuDukuChat_PiReady.html's Backend URL field at your deployed server (or hardcode it once you're ready).
  4. Complete Pioneer KYC if you haven't already, then register the app in the Pi Developer Portal (inside Pi Browser → Develop tab) using the pre-filled copy above and the icon/screenshots in listing_assets/.
  5. Complete domain verification, retake the 5 screenshots against your real deployment, and test the full flow on Testnet with Test-Pi.
  6. Submit for Mainnet review.
